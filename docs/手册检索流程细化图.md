# 手册检索流程细化图与代码说明

**版本**: 2026-05-14
**对应代码**: `src/modules/dual_route_retriever.py`（~1203 行）

---

## 1. 整体架构

```
用户问题
    │
    ▼
DualRouteRetriever.retrieve()           ← src/modules/dual_route_retriever.py:489
    │
    ├─→ route_query()                   # 路由决策：service / manual / mixed
    │       │
    │       ├─→ 拼写纠错（可选）         # LLMSpellCorrector.correct()
    │       ├─→ 规则路由评分             # _compute_rule_route_info()
    │       ├─→ ONNX 分类器预测         # RouteClassifier.predict()
    │       └─→ 规则+分类器裁决          # _arbitrate_route()
    │
    ├─→ 路由 == service
    │       └─→ _retrieve_service()     # 词项检索驱动（无向量索引依赖）
    │
    ├─→ 路由 == manual
    │       └─→ _retrieve_manual()     # FAISS 向量检索 + 多维评分重排
    │
    └─→ 路由 == mixed
            ├─→ _retrieve_service()     # top_k=4
            └─→ _retrieve_manual()      # top_k=5
                    │
                    └─→ _merge_results()   # service 优先，再按分数降序
```

## 2. 路由决策：`route_query()`

**位置**: `dual_route_retriever.py:212-286`

### 2.1 路由裁决优先级（`_arbitrate_route()`）

**位置**: `dual_route_retriever.py:426-475`

```
规则评分  +  分类器预测
        │
        ├── 优先级1: 强规则强制路由（无条件采纳）
        │
        ├── 优先级2: 分类器不可用 → 回退规则
        │
        ├── 优先级3: 高置信分类器
        │   (confidence ≥ 0.82  或  margin ≥ 0.25)
        │
        ├── 优先级4: 低置信分类器（回退规则）
        │   (confidence < 0.46  或  margin < 0.08)
        │
        ├── 优先级5: 分类器与规则一致 → 采纳分类器
        │
        ├── 优先级6: 仅分类器为 mixed → 升 mixed
        │
        ├── 优先级7: 规则偏好 service（无手册信号）
        │
        ├── 优先级8: 规则偏好 manual（有手册信号）
        │
        └── 优先级9: 其余情况 → 采纳规则
```

### 2.2 规则路由评分维度（`_compute_rule_route_info()`）

**位置**: `dual_route_retriever.py:288-393`

| 评分维度 | 权重 | 说明 |
|---------|------|------|
| `service_keyword_score` | `× 0.35` | 客服关键词命中数 / 4，归一化 |
| `service_example_score` | `× 0.65` | 与客服示例问题的词项相似度 |
| `manual_keyword_score` | `× 0.40` | 手册关键词命中数 / 3，归一化 |
| `manual_example_score` | `× 0.60` | 与手册示例问题的词项相似度 |
| `service_only_hits` 加分 | `+min(0.22, 0.08×n)` | 仅客服意图词命中时加 service 分 |
| 无手册信号时 service 加成 | `+0.12 / ×0.82` | 无候选手册且无手册关键词时降 manual 分 |
| 有图 + manual 关键词 | `+manual分` | 图片可能有助于手册理解 |

### 2.3 路由输出结构

```python
{
    "route": "service" | "manual" | "mixed",
    "classifier_label": str,
    "classifier_confidence": float,
    "classifier_margin": float,      # top1_prob - top2_prob
    "classifier_used": bool,         # 是否采纳了分类器
    "classifier_fallback_reason": str,
    "classifier_available": bool,
    "normalized_query": str,
    "manual_candidates": [(manual_name, alias_score), ...],
    "matched_intents": [intent, ...],
    "rule_service_score": float,
    "rule_manual_score": float,
}
```

## 3. Service 检索：`retrieve_service()`

**位置**: `dual_route_retriever.py:1020-1093`

### 3.1 检索流程

```
输入: query
        │
        ├── _match_service_intents(query)  ← 匹配客服意图关键词
        │   └─ 返回匹配到的 intent 列表
        │
        └── 对每条 service_documents 评分:
                ├─ score × 0.52           ← 查询与内容词项相似度
                ├─ title_score × 0.12     ← 查询与标题词项相似度
                ├─ keyword_bonus × 0.18   ← 意图关键词命中加分
                ├─ intent_bonus +0.18     ← 命中意图时额外加分
                ├─ doc_type_bonus         ← service_policy=0.12, playbook=0.05
                └─ priority × 权重        ← service_policy 优先级最高 0.82
        │
        └── 按 final_score 降序，取 top_k

输出: List[{content, doc_id, relevance_score, metadata: {route: "service"}}]
```

**特点**: Service 检索**不依赖向量索引**，全部基于词项相似度和意图匹配，适合 CPU 低资源环境。

### 3.2 关键词命中加分（`_intent_keyword_bonus()`）

```
命中数 / max(2, len(keywords) / 3)，归一化到 [0, 1]
```

例：意图有 6 个关键词，命中 2 个 → 2 / max(2, 2) = 1.0

## 4. Manual 检索：`retrieve_manual()`

**位置**: `dual_route_retriever.py:535-626`

### 4.1 整体流程

```
_retrieve_manual(query)
        │
        ├─→ 有候选手册?
        │       │
        │       ├─→ YES ─→ 取候选手册文档子集 candidate_docs
        │       │               │
        │       │               ├─→ 启用局部召回? ─→ YES ─→ 局部语义召回
        │       │               │       _local_semantic_recall()
        │       │               │               │
        │       │               │               ├─→ 检测到代码词? → YES → hybrid 检索
        │       │               │               └─→ 检测到代码词? → NO  → dense only
        │       │               │
        │       │               └─→ NO ─→ 全库检索 rag_engine.retrieve()
        │       │                       │
        │       │                       └─→ 多维评分重排 _rerank_manual_docs()
        │       │                               │
        │       │                               └─→  内部调用多样性惩罚
        │       │                                       _apply_diversity_penalty()  ← 内嵌在重排函数末尾
        │       │
        │       └─→ NO ─→ 全库检索 rag_engine.retrieve()
        │                       │
        │                       └─→ 多维评分重排 _rerank_manual_results()
        │                               │
        │                               └─→  内部调用多样性惩罚
        │                                       _apply_diversity_penalty()  ← 内嵌在重排函数末尾
        │
        └─→ 设置 metadata["route"] = "manual"，返回结果
```

**注意**: 多样性惩罚并非在 `_retrieve_manual` 末尾单独调用，
而是分别内嵌在 `_rerank_manual_docs()`（行 874）和 `_rerank_manual_results()`（行 837）的 return 语句中。
`_retrieve_manual()` 直接取这两个函数的返回值。

### 4.2 候选手册检测：`_detect_manual_candidates()`

**位置**: `dual_route_retriever.py:628-649`

```
输入: query
        │
        ├── 遍历 MANUAL_ALIAS_SEEDS 别名映射
        │   例: {"健身单车手册": ["健身单车", "动感单车", "单车"], ...}
        ├── 对每条别名检测是否在 query 中命中
        └── 计算别名得分 = sum(min(len(alias), 8) for alias in hits)
            └─ 长别名命中权重更高
        │
        └── 按得分降序返回候选手册列表

输出: [(manual_name, alias_score), ...]
```

**别名扩展**: `_build_manual_alias_map()` 自动从手册名本身提取别名：
- `"健身单车手册"` → `["健身单车", "动感单车", "单车", "健", "健身", ...]`
- 同时提取中英文混合词项

### 4.3 局部召回判断：`_should_use_local_manual_recall()`

**位置**: `dual_route_retriever.py:701-723`

| 条件 | 是否启用局部召回 |
|------|----------------|
| 最高分 ≥ 6.0（强阈值） | 是 |
| 仅 1 个候选且得分 ≥ 2.0 | 是 |
| 最高分 ≥ 2.0 **且** 与第二名分差 ≥ 2.0 | 是 |
| 其余情况 | 否（全库兜底） |

**意义**: 高置信时只检索目标手册子集，减少热门手册（如健身单车）造成的噪音；低置信时退全库召回保证召回率。

### 4.4 局部语义召回：`_local_semantic_recall()`

**位置**: `dual_route_retriever.py:725-770`

```
输入: query + candidate_docs（候选手册子集文档列表）
        │
        ├── 编码 query → BGE-M3 向量（1024 维）
        │   L2 归一化
        │
        ├── 对候选手册内每个文档：
        │   ├─ 从 FAISS 索引恢复向量（带缓存）
        │   │   └─ _get_manual_doc_vector()（行772-791）
        │   │       └─ 从 text_embeddings.reconstruct(idx) 恢复
        │   └─ 计算余弦相似度（点积）
        │
        ├── 按相似度降序
        └── 返回 top_k 个文档

缓存机制: manual_doc_vector_cache，避免重复恢复同一文档向量
```

### 4.5 全库召回：`rag_engine.retrieve()`

**位置**: `rag_engine.py`（`src/modules/rag_engine.py`）

```
输入: query, top_k, use_rerank
        │
        ├── 编码 query → BGE-M3 向量
        │
        ├── FAISS 索引搜索
        │   ├─ text_index.faiss: 文本 chunk 向量
        │   └─ image_index.faiss: 图片向量（当前默认未主动写入）
        │
        ├── 返回 top_k 个最近邻
        │
        └── use_rerank=True 时：
                ├─ BGE Reranker-v2-m3 交叉编码
                └─ 重排 top_k × rerank_candidate_k 个候选
```

### 4.6 多维评分重排：`_score_manual_result()`

**位置**: `dual_route_retriever.py:876-936`

| 评分维度 | 权重 | 说明 |
|---------|------|------|
| `semantic_score` | `× 0.34` | 向量语义相似度，最核心 |
| `content_score` | `× 0.26` | 查询词与内容文本的词项相似度 |
| `section_score` | `× 0.22` | 查询词与章节标题的相似度 |
| `alias_score` | `× 0.12` | 查询命中手册别名数量 |
| `candidate_scaled` | `× 0.12` | 手册候选加分（query 明确指定手册时） |
| `manual_name_score` | `× 0.06` | 查询与手册名的相似度 |
| `image_bonus` | `+0.06` | 含图文档额外加分（查询涉及图示类词汇时） |

**词项相似度计算**（`_lexical_similarity()`，行1019-1152）:

```
Jaccard × 0.45 + Coverage × 0.55

其中:
  Jaccard = |terms(query) ∩ terms(doc)| / |terms(query) ∪ terms(doc)|
  Coverage = |terms(query) ∩ terms(doc)| / max(1, |terms(query)|)

中文 2-gram 策略:
  "人体工学椅" → {"人体", "体工", "工学", "学椅", "人体工", "体工学", ...}
  英文/数字/下划线/连字符: 整体作为词项
  单字符中文: 直接作为词项
```

### 4.7 多样性惩罚：`_apply_diversity_penalty()`

**位置**: `dual_route_retriever.py:943-997`

对重排后 Top3 做多样性惩罚，防止同一章节的连续 chunk 全挤在前面：

| 命中场景 | 降权幅度 |
|---------|---------|
| Top2/Top3 与 Top1 同 `section_title` | `−0.05` |
| 同 `section_title` + 相邻 `chunk_index`（差 ≤ 2） | 叠加 `−0.03`（共 `−0.08`） |

**例外**: 同手册但不同章节不惩罚（保留步骤类问题的上下文连续性）。

## 5. 混合路由合并：`_merge_results()`

**位置**: `dual_route_retriever.py:1133-1172`

```
路由 == "mixed"
        │
        ├── service_results 取 top 4
        ├── manual_results 取 top 5
        ├── 合并为同一列表
        │
        └── 排序规则:
                主键: route == "service" → 排前（权重 1）
                次键: relevance_score 降序（权重 relevance_score）

结果: service 文档优先，同类按相关性分数排序
```

## 6. 关键数据结构

### 6.1 `DualRouteRetriever` 初始化数据

```python
# 手册别名映射（行 35-62）
MANUAL_ALIAS_SEEDS = {
    "健身单车手册": ["健身单车", "动感单车", "单车"],
    "健身追踪器手册": ["健身追踪器", "手环", "表带", "追踪器"],
    "VR头显手册": ["VR头显", "头显", "VR设备"],
    ...
}

# 客服意图规格（行 ~100+）
SERVICE_HINTS = ["退款", "退货", "换货", "发票", "保修", ...]
MANUAL_HINTS = ["说明书", "指示灯", "按钮", "如何", "安装", ...]

# 服务文档（来自 service_route_kb.json）
self.service_documents: List[Dict]    # 客服政策文档列表
self.service_kb_loaded: bool          # 是否已加载
self.intent_specs: Dict              # 意图规格含 keywords 列表

# 手册数据（来自 FAISS 索引）
self.manual_doc_map: Dict[manual_name -> List[Document]]
self.manual_alias_map: Dict[manual_name -> List[alias]]
self.manual_doc_index_map: Dict[doc_id -> faiss_index]
self.manual_doc_vector_cache: Dict    # 局部召回向量缓存
```

### 6.2 检索结果格式

```python
{
    "route_info": {...},           # 路由决策信息
    "service_results": [...],       # 客服检索结果
    "manual_results": [...],        # 手册检索结果
    "results": [...],              # 合并后的最终结果
}
```

每条检索结果：

```python
{
    "content": str,                 # 文档正文（可能含 <PIC>[image_id]）
    "doc_id": str,                  # 唯一标识
    "relevance_score": float,       # 综合相关性分数 [0, 1]
    "metadata": {
        "manual_name": str,          # 所属手册名
        "section_title": str,       # 章节标题
        "chunk_index": int,         # chunk 在章节中的索引
        "image_ids": List[str],     # 该 chunk 引用的图片 ID 列表
        "route": str,               # "service" 或 "manual"
    },
    "has_image": bool,              # 内容是否含图片引用
    "image_ids": List[str],         # 提取出的图片 ID 列表
}
```
