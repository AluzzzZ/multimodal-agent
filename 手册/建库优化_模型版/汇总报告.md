# 手册章节化与建库优化汇总报告

## 概述

使用模型 `deepseek-v4-pro` 对 20 本产品手册进行章节化与建库优化处理。

## 处理策略

**两阶段处理流程：**
1. **阶段1**：模型识别章节标题（极简 prompt，减少被过滤风险）
2. **阶段2**：从原文切分内容 + 模型规划合并策略

**PIC 绑定**：将原始 `<PIC>` 占位符与图片 ID 逐一绑定，确保图文对应关系完整保留。

## 已知限制

1. 部分手册（如健身追踪器）章节化节数较多（130 节），主要因为原手册包含大量参数规格表，建库版合并后块数合理（66 块）。
2. 发电机手册原始数据 PIC/ID 不一致，已做容错处理。
3. 部分模型对某些手册返回空输出（疑似内容安全过滤），通过备选 prompt 策略解决。
